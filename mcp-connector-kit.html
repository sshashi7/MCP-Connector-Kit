MCP Connector Kit
for Copilot Studio

Turn REST APIs into MCP Connectors for Copilot Studio.

What is this?

This is an interactive guide that helps you build and deploy an MCP (Model Context Protocol) server that connects any REST API to Microsoft Copilot Studio. It generates a ready-to-deploy TypeScript project — including server code, configuration files, and deployment instructions — so Copilot Studio agents can call your API as a tool.

MCP is an open standard that lets AI agents discover and invoke API capabilities dynamically. By wrapping your API in an MCP server, Copilot Studio agents can understand what your API offers and call it on behalf of users — with proper authentication, rate limiting, and error handling.

Prerequisites

Dev tools
  • ✅ Node.js 20 LTS or later — install
  • ✅ Azure CLI (az) — install
Azure & Copilot Studio
  • ✅ Azure subscription with permission to create App Services
  • ✅ Copilot Studio license (trial or paid)
  • ✅ A Copilot Studio agent to add the MCP tool to
Your API
  • ✅ API base URL — the root URL of the partner REST API you're connecting
  • ✅ API credentials — bearer token, API key, OAuth client ID/secret, or Entra ID app registration details (depends on auth type)
  • ✅ API documentation — endpoint paths, methods, parameters, and expected responses

End state

When you're done, you will have:

  • A TypeScript MCP server running on Azure App Service, wrapping your partner API
  • Secure authentication — inbound (Copilot Studio → your server) and outbound (your server → partner API)
  • A Copilot Studio agent that can call your API tools via natural language
  • Rate limiting, health checks, and structured error handling built in
  • A DEPLOYMENT.md with all commands needed to redeploy or hand off
⚠️ Scope: This kit is designed for PoC, demo, and internal use. Additional security hardening is recommended before production deployment.
